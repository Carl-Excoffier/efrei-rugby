## 1. Project Overview
This project is a bespoke web application built for the Efrei Rugby club to manage their schedule, roster, and player performance reviews. It is built using Node.js and Express, connected to a MySQL database, and hosted on an Amazon Web Services (AWS) EC2 Ubuntu instance.

## 2. Cloud Infrastructure Setup (AWS EC2)
The server is hosted on AWS EC2 using an Ubuntu AMI.

1. **Instance Creation:** Spun up a `t3.micro` Ubuntu instance.
2. **Security Groups:** Configured the AWS Security Group to allow inbound traffic on:
   * Port `22` (SSH for administration)
   * Port `80` (HTTP)
   * Port `443` (HTTPS for SSL/TLS)

## 3. Server Environment Configuration
*Note: The following commands should be run in the Ubuntu terminal via SSH.*

1. **Update the system:**
`sudo apt update && sudo apt upgrade -y`

2. **Install Node.js & npm (via Snap):**
`sudo snap install node --classic`

3. **Install MySQL Server:**
`sudo apt install mysql-server -y`

4. **Install PM2 (Process Manager):**
`sudo npm install -g pm2`

## 4. Database Setup
1. Log into MySQL: `sudo mysql -u root`
2. Create the database and dedicated user:
   ```sql

## 5. Application Deployment (Node.js & Nginx)

This application uses a robust, two-tier deployment strategy: **PM2** manages the Node.js backend to keep it running continuously, while **Nginx** acts as a reverse proxy to intercept public web traffic and route it securely to the application.

### Phase 1: Deploying the Node.js Backend
1. **Clone the repository** to the AWS server:
   `git clone [your-github-repo-link]`
2. **Navigate to the directory and install dependencies:**
   `cd [your-folder-name]`
   `npm install`
3. **Start the application with PM2:**
   PM2 is used to run the app in the background (on port 3000) and automatically restart it if the server crashes.
   `pm2 start server.js --name "efrei-rugby"`
4. **Configure PM2 to start on server boot:**
   `pm2 startup` *(Run the specific command generated by this output)*
   `pm2 save`

### Phase 2: Configuring the Nginx Reverse Proxy
By default, Node.js runs on port 3000, which is not exposed to the public internet. Nginx is configured to listen on port 80 (HTTP) and seamlessly forward that traffic to the Node.js app.

1. **Create a new Nginx configuration file:**
   `sudo nano /etc/nginx/sites-available/efrei-rugby`
2. **Add the reverse proxy block:**
   ```nginx
   server {
       listen 80;
       server_name [yourdomain.com];

       location / {
           proxy_pass http://localhost:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
   }
4. **Test the configuration for syntax errors**
  `sudo nginx -t`
5. **Restart Nginx to apply the changes**
  `sudo systemctl restart nginx`

## 7. Scripting Component: Dynamic Player Review Logic

**Objective:** The backend script (`server.js`) features a dynamic database routing script written in JavaScript (Node.js/Express). It is designed to simultaneously record a coach's performance review for a player and automatically recalculate that player's overall historical rating in real-time.

**The Code:**
```javascript
    app.post("/api/coach/add-review", requireCoachAPI, (req, res) => {
    const { playerId, eventId, rating, comment } = req.body;

    // Query 1: Insert the newly created review into the reviews table
    const insertQuery = `INSERT INTO reviews (player_id, event_id, rating, comment) VALUES (?, ?, ?, ?)`;

    db.query(insertQuery, [playerId, eventId, rating, comment], (err, results) => {
        if (err) {
            console.error(err);
            return res.json({ success: false, message: "Database error." });
        }

        // Query 2: Command MySQL to calculate the new average and update the player's main profile
        const updateAvgQuery = `
            UPDATE players
            SET avg_review = (SELECT ROUND(AVG(rating), 1) FROM reviews WHERE player_id = ?)
            WHERE id = ?
        `;
        
        db.query(updateAvgQuery, [playerId, playerId], (updateErr) => {
            if (updateErr) {
                console.error("Failed to update average:", updateErr);
                return res.json({ success: true, message: "Review posted, but average failed to update." });
            }
            res.json({ success: true, message: "Review posted & player average updated!" });
        });
    });
});
```

**Explanation:** This script executes when a coach submits a performance review form on the frontend. Rather than relying on the frontend to calculate statistics (which can lead to data desynchronization), this script securely chains two MySQL queries together on the backend.

First, it logs the individual review. Upon success, it immediately triggers a subquery that commands the MySQL engine to pull all historical reviews for that specific player_id, calculate the true average (`AVG`), round it to one decimal place (`ROUND`), and permanently update the `avg_review` column in the main players table.
